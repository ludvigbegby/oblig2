Git og Github

Git

Hovedforskjellen mellom Git og andre Version Control systemer, er hvordan den tenker på sin data. Git tenker på dataen mer som et sett med bilder av mini-filsystemer. Hver gang man commit-er, eller lagrer tilstanden for prosjektet ditt i Git, så tar den i hovedsak et bilde av hvordan alle filene dine ser ut i det øyeblikket og lagrer. For effektivitetens skyld, lagrer den ikke filer som ikke er blitt endret, men bare linker til forrige identiske fil som allerede er lagret.

De aller fleste operasjonene i Git trenger kun lokale filer og ressurser for å operere. I utgangspunktet trenger man ikke informasjon fra noen andre maskin på nettverket ditt nødvendig. På grunn av at man har hele historien til prosjektet på den lokale disken, virker de fleste operasjoner som om de skjer omtrent med en gang.

For å se gjennom historien til prosjektet ditt, trenger ikke Git å gå til serveren for å få den. Den viser det til deg fordi den leser rett fra lokale den database. Ønsker du derimot å se endringer gjennom hele historien, vil du få en detaljert oversikt over endringer og hvordan du endret dette. Git kan altså sjekke filer fra hver gang du lagret de, og utføre en lokal kalkulasjon av forskjellene istedenfor å måtte spørre en server om å gjøre det.

Når jeg forstår mer og mer hvor bra Git kan være, tror jeg at det er en utrolig ressurs for å kunne jobbe hvor som helst uten blant annet internett-tilgang. Det jeg synes er vanskelig er riktignok hvordan man bruker det enda. Jeg tror at den dagen jeg forstår Git utmerket, vil det være et ekstremt godt hjelpemiddel.

Github

Github er et nettsamfunn for programmerere. På facebook deler du bilder og statuser, på Github deler du koder. Det er et åpent miljø hvor programmerere kan fritt dele og samarbeide på åpen kildekode. Det gjør det enkelt å finne koden, kopiere til eget bruk og sende inn endringer til andres prosjekter. Github er rett og slett suverent for en bedrift, vennegjeng eller kollokviegruppe for å lage prosjekter sammen. En av mekanismene som er gode i Github er for eksempel å legge inn saker til prosjektet på Github, rapportere bugs eller be om funksjoner.
